# SkinCareAI 当前阶段摘要

更新日期：2026-04-12

## 当前阶段定位

项目当前处于“前端稳定化 + 本地知识承载层扩展 + 真实 API 接入初版”阶段。

这意味着：

- 聊天主流程已经稳定。
- 会话状态管理已经稳定。
- `/knowledge` 已从硬编码卡片升级为本地结构化知识包驱动。
- 已补齐真实 API 可接入的轻后端壳。
- 文本和图片理解路径已接入真实模型路由。
- 还没有进入 RAG、搜索和专家会诊阶段。

## 本轮主要增量

- 本地知识条目从 6 条扩到 21 条。
- 增加了 `manifest.json + import.meta.glob` 的自动汇总机制。
- 补了 `generate-knowledge-manifest.mjs` 生成脚本，并接入 `validate:knowledge`。
- `/knowledge` 页面升级为“精选 / 全部”两层结构，并补充分组显示。
- `source-registry` 增加了外部白名单候选来源登记与已使用外部文本来源登记。
- 病例图片内容层增加了更完整的类型、registry、占位 manifest 与条目关联。
- 已建立 `server/` mock 服务端壳，包含 health/models/chat/vision 接口。
- 已把 `server/` 推进为真实 provider adapter 架构，并保留 mock fallback。
- 前端模型列表已从真实 `/api/models` 获取。

## 当前最重要的成果

- 本地知识层现在已经具备：
- 结构化内容
- 来源登记
- manifest 聚合
- 基础校验
- 前端页面读取链路
- 服务端接口壳体
- 真实文本与视觉接口通路

## 仍未进入的阶段

- 检索增强
- 搜索框
- 引用卡片

- 补充病例图片最小演示集接入。

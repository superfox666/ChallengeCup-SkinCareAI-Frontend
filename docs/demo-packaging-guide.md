# SkinCareAI 演示与录屏准备指南

更新日期：2026-04-15

这份文档面向演示者、队长或接手录屏的人。重点是“怎么讲最稳、先讲什么、哪些镜头要避开”。

## 1. 建议演示顺序

### 第一段：主链路开场

1. 打开 `/chat`
2. 先讲这是皮肤健康咨询网页，不讲历史开发过程
3. 指向默认模型 `Qwen 3 VL Flash`
4. 直接发起一次文本问诊

### 第二段：图片链路

1. 上传一张 demo 图片
2. 演示图片预览
3. 演示图文问诊回复

### 第三段：交互和管理

1. 演示新建对话
2. 演示会话搜索
3. 演示时间分组
4. 演示重命名 / 删除
5. 演示复制回答、复制当前会话、导出 Markdown

### 第四段：加分项

1. 演示非视觉模型 handoff
2. 演示轻量专家会诊
3. 切到 `/knowledge`
4. 展示真实病例图区块

## 2. 哪些功能先讲

- 文本问诊
- 图片问诊
- 加载态
- 多轮历史
- 模型工作台

这些是第一层，优先级最高。

## 3. 哪些功能后讲

- 会话搜索
- 时间分组
- 重命名 / 删除
- 主题切换
- `/knowledge`
- 轻量专家会诊

这些是增强项，放在主链路之后讲。

## 4. 哪些地方适合录视频

- 聊天页深色主题桌面
- 文本问诊完整链路
- 图片上传后图文回复
- 非视觉模型 handoff
- 轻量专家会诊展开
- `/knowledge` 桌面页首屏

## 5. 哪些功能暂时不要录或谨慎录

- `系统分享`
  - 没有真机确认前不要录进最终成片。
- `/knowledge` 手机竖屏长滚动
  - 这页在手机上只适合录首屏和关键展开。
- `qwen-plus-latest` 主文本问诊
  - 不要把它录成主入口模型。

## 6. 主演示模型用什么

### 默认主模型

- `Qwen 3 VL Flash`

理由：

- 文本和图片都能讲
- 当前主路径最稳
- 国内链路更适合现场演示

### 文本备选

- `GPT-5.4 mini`
- `Qwen 3.6 Plus`

### 深度推理补充

- `DeepSeek R1`

### 不推荐主演示

- `qwen-plus-latest`

## 7. 若 API 波动如何兜底

- 当前后端保留 mock fallback。
- 如果真实链路临时波动，页面仍可继续走完整交互流程。
- 讲法不要说“所有请求都稳定无波动”，而是说“真实 API 已接入，同时保留演示兜底能力”。

## 8. 录屏前检查

1. 浏览器缩放确认 `100%`
2. 前后端都已启动
3. 聊天页能正常打开
4. `/api/health` 返回正常
5. `/api/models` 返回正常
6. 默认模型显示为 `Qwen 3 VL Flash`
7. 示例图片路径可访问
8. 如果要录系统分享，先拿真机确认原生面板

## 9. 当前最稳的录屏脚本

1. 打开聊天页深色主题
2. 文本问诊一轮
3. 图片问诊一轮
4. 演示 handoff
5. 演示会话搜索和导出
6. 演示轻量专家会诊
7. 打开 `/knowledge` 桌面首屏
8. 停在病例图区块结束

## 10. 当前不建议说满的话

- 不要说系统分享已经全部闭环。
- 不要说已经建成完整真实病例图库。
- 不要把 `qwen-plus-latest` 说成主文本模型。

## 11. 当前最合适的结束口径

- 主链路已经锁定
- 文本、图片、知识页和会诊都可演示
- 真实 API 已接入
- 首批真实病例图已接入
- 系统分享真机确认仍是最后一个外部检查项

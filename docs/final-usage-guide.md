# SkinCareAI 使用指南

更新日期：2026-04-15

这份文档面向“第一次接手项目的人”。重点不是讲开发过程，而是讲清楚这个项目现在能做什么、页面怎么用、哪些功能值得展示。

## 1. 页面说明

### `/chat`

这是主页面，也是最推荐的演示主路径。

页面主要分成三块：

- 左侧：会话区
- 中间：聊天主区
- 右侧：辅助说明区

### `/knowledge`

这是结构化知识页，用于展示固定知识卡片和病例图演示集。

适合用来讲：

- 项目不仅能问答，也有静态知识展示
- 病例图来源、目录、清单和接入状态
- 中医 / 西医 / 中西医结合三类内容组织方式

不适合用来讲：

- 长时间手机竖屏整页滚动
- 复杂检索系统
- RAG 能力

## 2. 聊天页各部件做什么

### 左侧品牌区

- 用来说明这是当前工作区，不是装饰区。
- 主要作用是稳定整体结构和识别项目品牌。

### 左侧会话列表

- 展示当前本地会话历史。
- 支持搜索、时间分组、切换、重命名、删除。

### 顶部主标题区

- 显示当前会话标题和当前会话类型。
- 会在聊天页和知识页之间切换导航。

### 模型工作台

- 显示当前模型状态、推荐入口和模型选择器。
- 这里强调“先用默认模型，不确定再切换”。

### 帮助问号

- 出现在模型、主题、图片上传、知识页等认知成本较高的位置。
- 桌面端是近场说明，移动端会改成底部弹层。

### 聊天消息区

- 清晰区分用户消息和 AI 回复。
- 支持多轮上下文展示。
- AI 回复生成时会出现明确加载状态。

### 输入区

- 支持纯文本输入。
- 支持上传、拖拽、粘贴图片。
- 图片会在发送前先出现在预览区。

### 右 rail

- 显示当前模型能力、当前使用边界和简短说明。
- 这里不是第二个文档页，只是辅助讲解区。

## 3. 每个功能怎么用

### 新建对话

- 左侧点击“新建对话”。
- 会清空当前上下文，开始新会话。

### 模型切换

- 在顶部模型工作台打开模型选择器。
- 切换模型会自动进入新会话，不混旧上下文。

### 文本问诊

1. 在输入框中输入皮肤相关问题。
2. 按 `Enter` 发送。
3. 等待流式回复完成。

### 图片问诊

1. 上传、拖拽或粘贴图片。
2. 补一条简短问题。
3. 发送后查看图文联合回复。

### 非视觉模型 handoff

1. 先切到不支持图片的文本模型。
2. 再上传图片。
3. 系统会自动建议并切换到视觉模型的新会话。

### 会话搜索

- 在左侧搜索框输入标题、模型名或问诊类型关键词。
- 列表会即时收缩。

### 会话时间分组

- 会话会按“今天 / 昨天 / 近 7 天 / 更早”自动分组。

### 重命名 / 删除

- 在单条会话卡片上操作。
- 重命名用于把问诊主题讲得更清楚。
- 删除用于清理无用演示分支。

### 复制回答

- 在 AI 回复旁点击复制按钮。
- 适合快速拷贝单条结果。

### 复制当前会话

- 在顶部会话动作区触发。
- 会复制当前会话快照文本。

### 导出 Markdown

- 在顶部会话动作区触发。
- 适合留存答辩材料或对外发文档。

### 系统分享

- 只在支持 `navigator.share` 的设备上显示。
- 建议只在真实移动设备上演示。

### 主题切换

- 顶部可切换 `dark` / `light` / `soft`。
- 推荐答辩默认用深色。

### `/knowledge`

- 顶部先看精选与全部。
- 再切分类。
- 最后看病例图区块和使用说明。

### 轻量专家会诊

- 在聊天页打开会诊区。
- 展开后会自动发起一轮并排比较。

## 4. 哪些地方值得在答辩里讲

### 基础功能

- 文本问诊
- 图片问诊
- 消息流
- 多轮历史
- 加载态

### 体验增强

- 新建对话
- 会话搜索
- 时间分组
- 重命名 / 删除
- 主题切换
- 粘贴 / 拖拽上传
- 非视觉模型 handoff

### 展示加分项

- `/knowledge`
- 轻量专家会诊
- 模型工作台
- 模型分层与推荐
- 帮助问号和近场提示

### 当前特色

- 一个模型一个会话
- 会话与模型绑定
- 默认推荐模型
- 图文联合输入
- 真实 API + fallback

## 5. 当前推荐模型

### 主演示模型

- `Qwen 3 VL Flash`
- `GPT-5.4 mini`
- `Qwen 3.6 Plus`

### 备选模型

- `DeepSeek R1`

### 不建议走主演示路径

- `qwen-plus-latest`

原因：

- 它保留为低成本中文备选
- 历史上出现过超时 fallback
- 不适合作为现场主演示文本模型

# SkinCareAI

SkinCareAI 是一个面向皮肤健康咨询、图文问诊、结构化知识科普和多模型协作的 Web 工作区项目。本仓库在原有项目基础上补全了 `web/` 前端工作区，形成了可运行、可演示、可交接的完整版本。

## 项目背景

本项目的目标不是只做一个聊天壳，而是把皮肤问诊场景下的几条核心链路打通：

- 文本问诊
- 图片问诊
- 多轮会话管理
- 多模型切换与连通性监测
- 结构化知识卡片展示
- 专家会诊与结果导出

当前版本的整体设计固定为：

- 顶部栏 + 工作区标签页 + 左上角覆盖式抽屉
- 聊天主区优先，低频说明后置
- 模型工作台与知识页作为辅助工作区
- 问号提示、弹层说明、主题切换、全屏模式共同增强演示体验

## 快速开始

### 1. 启动后端

```powershell
cd server
npm install
copy .env.example .env
npm run dev
```

后端默认地址：

- `http://127.0.0.1:8787`

若出现 `EADDRINUSE: address already in use 127.0.0.1:8787`，说明本机已有后端实例正在监听该端口。排查步骤如下：

```powershell
Get-NetTCPConnection -State Listen -LocalPort 8787
Get-CimInstance Win32_Process -Filter "ProcessId=<OwningProcess>"
```

如需结束旧实例：

```powershell
Stop-Process -Id <OwningProcess> -Force
```

### 2. 启动前端

```powershell
cd web
npm install
npm run dev
```

前端默认地址：

- `http://127.0.0.1:5173`

当前前端开发服务器已固定：

- `host = 127.0.0.1`
- `port = 5173`
- `strictPort = true`

这意味着 `5173` 被占用时会直接报错退出，不会静默漂移到其他端口。

### 3. 构建与检查

```powershell
cd web
npm run build
npm run lint
```

当前仓库最近一次实测结果：

- `npm run build` 通过
- `npm run lint` 通过

## 运行实例核对

在浏览器打开页面后，建议先核对当前是否为本仓库的最新运行实例，再开始演示或交接。

核对项如下：

1. `http://127.0.0.1:5173/chat` 顶部可见 `聊天工作区 / 知识页`
2. `http://127.0.0.1:5173/knowledge` 首屏可见完整知识卡片区
3. 模型工作台可见模型总数、最近检测时间和 `连通性测试`
4. 浏览器控制台可见 `[SkinCareAI runtime]` 调试标识

若页面表现与源码不一致，优先检查：

- 是否误开了旧的 `vite` 实例
- 是否仍在复用旧的 `server` 实例
- 是否打开了错误端口
- 是否复用了旧标签页而非当前实例

## 页面与使用说明

### 1. 主工作区结构

主界面固定采用三层结构：

- 顶栏：品牌、主导航、主题切换、帮助、更多工具
- 工作区标签页：`聊天`、`模型`
- 主内容区：根据当前标签切换聊天页或模型工作台

左上角抽屉作为统一低频入口，负责承接：

- 新建会话
- 会话搜索
- 历史会话列表
- 标签与归档
- 帮助与免责声明
- 产品介绍与使用方式

### 2. 聊天页

聊天页是主路径，结构为：

- 极短状态条
- 消息展示区
- 输入与上传区

聊天页支持的关键能力：

- 文本问诊
- 图片上传、拖拽、粘贴
- 图片预览
- 生成中动画
- 回复语气高级设置
- 单条 AI 回答导出 Markdown
- 聊天全屏

当前实现中，聊天页还支持以下体验细节：

- 切换模型时保留当前文字与图片草稿
- 非视觉模型收到图片时自动 handoff
- 输入区压缩后仍保持消息区为首屏最大主体

### 3. 模型工作台

模型工作台位于 `模型` 标签页，当前包含：

- 模型分类条
- 模型列表区
- 模型能力说明区
- 延迟与状态展示
- 最近检测时间
- 手动 `连通性测试`
- 模型全屏按钮
- 全屏状态下的 `去咨询` 按钮

模型工作台的定位不是静态说明页，而是可直接完成以下动作：

- 比较不同模型
- 观察在线状态与延迟
- 选择目标模型
- 一步切回聊天页发起问诊

### 4. 知识页

`/knowledge` 页面用于结构化展示固定知识内容，当前保留：

- 知识分类
- 主知识卡片区
- 导览与阅读提示
- 病例图演示内容

知识页的定位是：

- 用于答辩演示中的结构化讲解
- 用于体现系统不只具备聊天能力，也具备知识展示能力
- 用于承接皮肤知识、病例示意和主题分类

### 5. 专家会诊

专家会诊入口位于顶栏的 `更多工具` 与相关主区入口中。该功能用于：

- 围绕当前问题并排比较多个模型
- 汇总各模型观点
- 形成更适合展示的多专家讨论结果

### 6. 导出与分享

当前支持两类导出：

- 整段会话导出：顶栏复制、导出 Markdown、原生分享
- 单条回答导出：AI 回复右侧图标导出 Markdown

## 体验设计与实现细节

当前版本重点优化了以下体验点：

- 顶栏缩高，减少首屏垂直占用
- Logo 放大，强化左上角品牌锚点
- 工作区标签页轻量化，避免重复信息堆叠
- 抽屉采用覆盖式结构，不挤压主内容宽度
- 问号帮助采用近场提示，不让说明长期占据主区
- 聊天页与模型页分别支持全屏，适合录屏演示
- 模型页全屏加入 `去咨询`，减少演示跳转步骤
- 主题切换、免责声明、帮助入口统一收口到顶栏
- 模型延迟与连通性检测常驻可见，便于展示模型状态
- 切换模型时保留草稿，降低问诊中断成本

适合在演示中重点强调的设计细节包括：

- 侧边抽屉统一承接会话与说明入口
- 工作区标签页降低导航复杂度
- 顶栏问号提示与低频说明弹层化
- 单条 AI 回答右侧导出按钮
- 模型工作台的全屏与 `去咨询`
- 主题切换、免责声明短提示、品牌 Logo 强化
